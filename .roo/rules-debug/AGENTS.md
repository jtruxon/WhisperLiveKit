# Debug Mode Rules — WhisperLiveKit (Non-Obvious Only)

Read [`AGENTS.md`](../../AGENTS.md:1) first. These are deltas for debugging tasks.

- **Always start with `uv run wlk diagnose <audio.wav>`.** It feeds a real file through the full pipeline and probes backend internals at intervals — exposes problems that `wlk serve` will hide. See [`docs/API.md`](../../docs/API.md:170).
- **Pipeline state is on three `asyncio.Queue`s**: `transcription_queue`, `diarization_queue`, `translation_queue` ([`audio_processor.py:116-118`](../../whisperlivekit/audio_processor.py:116)). End-of-stream is a `SENTINEL` object pushed via `_signal_input_complete` ([`audio_processor.py:290`](../../whisperlivekit/audio_processor.py:290)) into the **first active queue only** and chained downstream. If a downstream task hangs at shutdown, the SENTINEL was probably swallowed by an exception upstream.
- **Watchdog runs every 10s** and logs `"unexpectedly completed with exception"` ([`audio_processor.py:644`](../../whisperlivekit/audio_processor.py:644)) — grep for this first when sessions stall. Stuck transcription with non-empty queue ⇒ usually a deadlock on the global model lock or a bug inside `process_iter()`.
- **FFmpeg failure modes** ([`ffmpeg_manager.py`](../../whisperlivekit/ffmpeg_manager.py:39)): five states (`STOPPED, STARTING, RUNNING, RESTARTING, FAILED`) and five callback strings (`ffmpeg_not_found`, `start_failed`, `write_error`, `read_error`, `restart_failed`). These flow up to the client as `FrontData.error`. `read_data` has a 20s timeout ([`ffmpeg_manager.py:152`](../../whisperlivekit/ffmpeg_manager.py:152)) — if the client sends silence for >20s, expect `read_error`.
- **SimulStreaming repetition / hallucination loops** are bounded by constants in [`simul_whisper/backend.py:42-46`](../../whisperlivekit/simul_whisper/backend.py:42): `_INTRA_BATCH_REWIND_SECONDS=0.75`, `_REWIND_RESET_SECONDS=1.0`, `_RECENT_WORD_HISTORY=80`, `_MIN_REPETITION_WORDS=12`. Tune here, not in callers. Regression tests live at [`tests/test_backend_deep_bugs.py:182-298`](../../tests/test_backend_deep_bugs.py:182).
- **Thread-safety env vars**: `WHISPERLIVEKIT_MODEL_LOCK=1` (default) serializes all model inference per-process; `WHISPERLIVEKIT_LOCK_TIMEOUT` defaults to 30s ([`thread_safety.py:30-31`](../../whisperlivekit/thread_safety.py:30)). Set lock to `0` only for single-connection deployments. Lock contention symptoms: timeouts in logs + sessions appearing to "take turns".
- **`TranscriptionEngine.reset()` is destructive** ([`core.py:52`](../../whisperlivekit/core.py:52)) — only call it inside `try/finally` in tests. Calling it while sessions exist crashes them silently.
- **MLX/Voxtral/Qwen3 backends are silently skipped** in tests when their packages aren't importable — there is **no `pytest.skip` message**. If a test "passes" suspiciously fast, check `tests/test_pipeline.py::AVAILABLE_BACKENDS` to confirm it actually ran for your target backend.
- **Qwen3 vLLM Metal pins ALL MLX work to one thread** via [`_Qwen3MetalWorker`](../../whisperlivekit/qwen3_vllm_metal_asr.py:55). Calling its model from any other thread yields cryptic Metal errors — wrap calls through the worker's submit interface.
- **Without `onnxruntime` installed**, the JIT Silero VAD model loads **per-session** (slow, large memory footprint) — a warning is logged at startup ([`core.py:104-114`](../../whisperlivekit/core.py:104)). For multi-user issues, check this first.
- **The OpenAI REST adapter shells out to `ffmpeg` directly** ([`basic_server._convert_to_pcm`](../../whisperlivekit/basic_server.py:145)), separate from `FFmpegManager`. Requires `ffmpeg` on PATH — failures here look like REST 500s, not WebSocket errors.
- **Diff-protocol desync**: if the client logs `len(lines) != n_lines`, the recommendation is to reconnect ([`docs/API.md:520`](../../docs/API.md:520)). The protocol does not heal automatically.
