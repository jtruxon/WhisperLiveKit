# Ask Mode Rules — WhisperLiveKit (Non-Obvious Only)

Read [`AGENTS.md`](../../AGENTS.md:1) first. These are deltas for answering documentation/explanation questions about this codebase.

- **[`docs/API.md`](../../docs/API.md:1) is the canonical wire-protocol reference** — point users there for any WebSocket / REST / Deepgram contract question, not at the source. It documents both `?mode=full` (default, no `type` field) and `?mode=diff` (snapshot + incremental) message schemas.
- **There are 3 streaming "policies" but 6 "backends"** — and they cross-cut. **Policies** = `simulstreaming` (AlignAtt) vs `localagreement` (HypothesisBuffer) vs each generative backend's own native streaming (Voxtral, Qwen3). **Backends** = whisper, faster-whisper, mlx-whisper, voxtral-mlx, voxtral-hf, qwen3-vllm(-metal). Decision tree lives in [`core.py:138-199`](../../whisperlivekit/core.py:138). Clarify which dimension a user is asking about.
- **`--backend auto` only chooses among Whisper-family backends** (mlx-whisper / faster-whisper / whisper) — see [`whisper_online.py:172-178`](../../whisperlivekit/local_agreement/whisper_online.py:172). Voxtral and Qwen3 must be selected explicitly. Users hitting "auto didn't pick my backend" need to know this.
- **The vendored Whisper at [`whisperlivekit/whisper/`](../../whisperlivekit/whisper/__init__.py:1) is NOT PyPI's `openai-whisper`** — it's a fork (version `20250625`) modified to support hookless decoding for SimulStreaming. Imports are `from whisperlivekit.whisper import …`. Don't direct users to upstream Whisper docs for our internals.
- **Many extras are mutually exclusive** ([`pyproject.toml:84-109`](../../pyproject.toml:84) `[tool.uv.conflicts]`): e.g. `voxtral-hf` ⊥ `diarization-sortformer`, `qwen3-vllm` ⊥ `cu129`, `cpu` ⊥ `cu129`. Direct users here before they try `uv sync --extra voxtral-hf --extra diarization-sortformer` and get confused.
- **Default model = `base`**, cache dir = `~/.cache/whisper` ([`docs/default_and_custom_models.md`](../../docs/default_and_custom_models.md:1)). Downloads come from `openaipublic.azureedge.net` (vendored Whisper), HuggingFace (faster-whisper, MLX, Voxtral, Qwen3), or local paths.
- **Speaker `-2` in output = silence**, not a speaker ID — threshold = 5 seconds (`MIN_DURATION_REAL_SILENCE` in [`audio_processor.py:26`](../../whisperlivekit/audio_processor.py:26) and [`simul_whisper/backend.py:37`](../../whisperlivekit/simul_whisper/backend.py:37)). Spec is in [`docs/API.md:526-540`](../../docs/API.md:526).
- **`.en` model suffix forces English** regardless of `--language` flag ([`config.py:97-99`](../../whisperlivekit/config.py:97)). Catches users who expect `tiny.en` + `--language fr` to work.
- **Embedding without the FastAPI server** is documented in [`docs/technical_integration.md`](../../docs/technical_integration.md:1) — minimal API is `AudioProcessor.process_audio(bytes)` in, async generator of `FrontData` out, `cleanup()` at end.
- **GPU install issues**: send users to [`docs/troubleshooting.md`](../../docs/troubleshooting.md:1) — covers cuDNN visibility, CTranslate2 CPU-vs-CUDA wheel pitfalls, NVIDIA Hopper/Blackwell `sm_121a` Triton workaround.
- **Diarization alignment internals** are in [`docs/alignement_principles.md`](../../docs/alignement_principles.md:1) (note misspelling) — ASCII diagrams explain how STT punctuation aligns with diarization speaker-change events. Reference for understanding `tokens_alignment.py`.
- **`DEV_NOTES.md`** ([`DEV_NOTES.md`](../../DEV_NOTES.md:1)) contains internal R&D memos: M4 encoder benchmarks justifying the hybrid PyTorch-decoder + faster-encoder design, NLLB Transformers vs CTranslate2 benchmarks (Transformers wins on macOS — drives the default), and the SortFormer 4→2 speaker-constraint algorithm. Not in the public docs but useful context for "why is it built this way?" questions.
- **CI runs no tests** — only `ruff check` + an import smoke test ([`.github/workflows/ci.yml`](../../.github/workflows/ci.yml:1)). A green CI badge does NOT mean tests pass.
