# Architect Mode Rules — WhisperLiveKit (Non-Obvious Only)

Read [`AGENTS.md`](../../AGENTS.md:1) and [`CLAUDE.md`](../../CLAUDE.md:1) first. These are deltas for design / planning tasks.

- **The embedding contract is intentionally tiny**: `AudioProcessor.process_audio(bytes)` in, async generator of `FrontData` out, `cleanup()` at end ([`docs/technical_integration.md`](../../docs/technical_integration.md:1)). Designs that introduce additional public APIs should be questioned — most needs are met by composing existing primitives.
- **`WhisperLiveKitConfig`** ([`config.py:18`](../../whisperlivekit/config.py:18)) is the **single source of truth** for runtime configuration. New options must (a) be a dataclass field with a default, (b) work via both `from_namespace()` (CLI) and `from_kwargs()` (programmatic embedding). Don't propose a parallel config dict, env-var registry, or YAML loader — they would fragment the source of truth.
- **Singleton `TranscriptionEngine` is a deliberate trade-off** ([`core.py:37`](../../whisperlivekit/core.py:37)): one set of heavy models is shared across all sessions in a worker, but model inference is sequential within the process. **Multi-tenant scaling = multiple workers**, each with its own singleton, behind nginx/gunicorn ([`README.md:298-307`](../../README.md:298), [`thread_safety.py:104-135`](../../whisperlivekit/thread_safety.py:104)). Designs proposing in-process concurrency (e.g. multiple engines, model pools) need to justify why the worker model is insufficient.
- **Two streaming abstractions co-exist** and should not be conflated:
  - **Chunk-based** (Whisper-family backends): only emit when `process_iter()` completes a chunk.
  - **Native-streaming** (Voxtral, Qwen3): emit tokens as they arrive from `model.generate()`.
  - The taxonomy is canonical in [`BACKENDS[].streaming`](../../whisperlivekit/cli.py:52) (`chunk` / `native` / `cloud`). Features that depend on token cadence (live captions, partial highlighting) MUST specify which class is in scope.
- **Diff-protocol opt-in is a designed trade-off** — full mode is the default because the client logic is trivial. Don't propose making diff the default without redesigning the reconstruction guarantees and the desync-recovery story ([`docs/API.md:478-522`](../../docs/API.md:478), [`diff_protocol.py:39-99`](../../whisperlivekit/diff_protocol.py:39)). Today, desync = client must reconnect.
- **Vendored Whisper is a hard fork** at [`whisperlivekit/whisper/`](../../whisperlivekit/whisper/__init__.py:1). Proposals to revert to PyPI `openai-whisper` must account for hookless-decoder modifications enabling shared model weights with per-session [`DecoderState`](../../whisperlivekit/simul_whisper/decoder_state.py:1). This is non-trivial — losing this enables per-session memory waste at scale.
- **Optional-extras conflict matrix is part of the architecture** ([`pyproject.toml:84-109`](../../pyproject.toml:84) `[tool.uv.conflicts]`):
  - `voxtral-hf` ⊥ `diarization-sortformer` (incompatible torch/transformers).
  - `qwen3-vllm` ⊥ `cu129` (vLLM bundles its own torch).
  - `cpu` ⊥ `cu129`.
  - `voxtral-mlx` / `qwen3-vllm-metal` / `mlx-whisper` are macOS-arm64 only (Apple Silicon).
  New backend designs must explicitly state where they fit in this matrix and which existing extras they conflict with.
- **`engine.config` AND `engine.args` are both kept on purpose** ([`core.py:94-97`](../../whisperlivekit/core.py:94)) — `args = Namespace(**asdict(config))` is exposed for backward compat with `AudioProcessor`. Removing `args` would be a breaking change to embedders. Refactors must preserve both.
- **Per-session state lives in [`SessionASRProxy`](../../whisperlivekit/session_asr_proxy.py:10) wrapping a shared ASR singleton.** Extending session state (per-session prompts, glossaries, custom vocab) should follow this pattern: lazily-attached lock + getattr delegation. Don't propose per-session ASR instances — defeats the singleton.
- **Three asyncio queues + watchdog model** ([`audio_processor.py:116-118,644`](../../whisperlivekit/audio_processor.py:116)) is the concurrency design. Proposals to add new pipeline stages (e.g. punctuation-restoration, profanity-filtering) should be a fourth queue + processor task, not inline mutations of `results_formatter`.
- **`vllm-metal` ships as a pinned `.whl` URL** ([`pyproject.toml:112`](../../pyproject.toml:112)) — Python 3.12 + Apple Silicon only. Designs that require vLLM on macOS need to plan around this constraint (no aarch64 Linux, no other Python versions).
- **The bundled web client is inlined into a single `/` HTML response** ([`web_interface.py:16`](../../whisperlivekit/web/web_interface.py:16)) and rewrites worker URLs to Blob URLs at serve time. Proposals to externalize JS / use a build step / split into multiple endpoints would break this contract — the substring rewriting is intentional and depends on exact strings (`/web/pcm_worklet.js`, `/web/recorder_worker.js`).
- **No formatter / no test-running CI** ([`.github/workflows/ci.yml`](../../.github/workflows/ci.yml:1)) is a deliberate stance — `ruff check` + import smoke only. Proposals to add black/isort/mypy or to gate merges on the test suite should be raised explicitly, not slipped in.
